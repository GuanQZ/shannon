# 中文渗透测试监控仪表板设计文档

**项目**：Shannon 渗透测试框架
**日期**：2026-03-02
**功能**：中文实时监控仪表板

## 1. 概述

本文档描述了为 Shannon AI 渗透测试框架设计的中文实时监控仪表板。该仪表板旨在帮助中文用户实时监控渗透测试进度，通过直观的可视化界面展示工作流执行状态。

### 1.1 设计目标

- **可读性**：全中文界面，方便中文用户理解当前进度
- **实时性**：实时显示日志输出和阶段进度
- **交互性**：支持按 Agent 筛选日志，过滤器支持开关切换

### 1.2 技术选型

采用单页面 HTML 应用实现，无需复杂后端，直接读取 Temporal 服务或本地日志文件。

## 2. 界面设计

### 2.1 整体布局

采用三栏式布局，从左到右依次为：阶段进度栏 → 实时日志栏 → Agent 列表栏。

```
┌─────────────────────────────────────────────────────────────────────┐
│  Shannon 渗透测试监控                                               │
├────────────┬────────────────────────────────────┬───────────────────┤
│            │                                    │                   │
│  阶段进度  │         实时日志                   │    Agent 列表     │
│            │                                    │                   │
│  ● Pre-Recon │ [pre-recon] 正在执行子域名扫描   │  ▸ Pre-Recon     │
│  ● Recon    │ [pre-recon] 识别到 15 个端点      │    pre-recon     │
│  ● Vuln    │ [recon] 开始端口扫描               │  ▸ Recon         │
│  ● Exploit │                                    │    recon          │
│  ● Report  │                                    │  ▸ Vulnerability │
│            │                                    │    vuln-injection│
│            │                                    │    vuln-xss      │
│            │                                    │  ▸ Exploitation  │
│            │                                    │  ▸ Reporting     │
│            │                                    │                   │
└────────────┴────────────────────────────────────┴───────────────────┘
```

### 2.2 左侧：阶段进度条

**功能**：显示五个渗透测试阶段的执行状态

**阶段列表**：

1. **Pre-Recon**（信息收集前置）
2. **Recon**（侦察）
3. **Vulnerability**（漏洞分析）
4. **Exploitation**（漏洞利用）
5. **Reporting**（报告生成）

**视觉设计**：

- 每个阶段用圆形标记表示状态
- 状态颜色语义：
  - 灰色：未开始
  - 黄色闪烁：执行中
  - 绿色：已完成
  - 红色：失败
- 阶段名称使用中文
- 当前阶段显示"执行中"文字提示

**布局参数**：

- 宽度占比：约 10%
- 垂直排列，居中对齐

### 2.3 中间：实时日志栏

**功能**：显示所有 Agent 的实时执行日志

**日志格式**：

```
[agent-name] 日志内容
```

其中 `agent-name` 为 Agent 名称前缀，用于区分不同 Agent 的输出。

**功能特性**：

- **自动滚动**：新日志自动滚动到底部
- **中文翻译**：日志内容保持原样（已为中文）
- **AI 思考过程**：显示 AI 推理过程（thinking 内容）
- **时间戳**：每行日志可选择显示时间戳
- **颜色编码**：不同级别日志用不同颜色（info: 白色，warning: 黄色，error: 红色）

**数据源**：

- 方案一：轮询 Temporal 查询接口获取进度
- 方案二：直接读取 `audit-logs/` 目录下的 JSON 日志文件

**布局参数**：

- 宽度占比：约 60%
- 最大高度：视窗高度减去标题栏
- 滚动：支持鼠标滚轮和滚动条

### 2.4 右侧：Agent 列表栏

**功能**：按阶段分组的 Agent 列表，支持点击筛选

**分组结构**：

```
▸ Pre-Recon (1)
    pre-recon
▸ Recon (1)
    recon
▼ Vulnerability (5)
    vuln-injection ✓
    vuln-xss
    vuln-auth
    vuln-authz
    vuln-ssrf
▸ Exploitation (5)
▸ Reporting (1)
```

**交互设计**：

- 点击阶段名称：折叠/展开该阶段下的 Agent 列表
- 点击 Agent 名称：筛选日志，只显示该 Agent 的日志
- 再次点击同一 Agent：取消筛选，显示所有日志
- 选中状态：Agent 名称高亮显示
- 完成标记：已完成的 Agent 显示 ✓ 绿色勾选

**布局参数**：

- 宽度占比：约 30%
- 每个 Agent 占用一行
- 折叠时只显示阶段名和数量

## 3. 功能规格

### 3.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 阶段进度显示 | 实时显示 5 个阶段的执行状态 | 必选 |
| Agent 列表显示 | 按阶段分组显示 13 个 Agent | 必选 |
| 日志实时显示 | 实时输出日志内容 | 必选 |
| Agent 筛选 | 点击 Agent 筛选对应日志 | 必选 |
| 筛选切换 | 重复点击取消/启用筛选 | 必选 |
| AI 思考显示 | 显示 AI 推理过程 | 可选 |

### 3.2 数据交互

**进度数据获取**：

- 使用 Temporal Query 接口查询工作流进度
- 查询端点：`/temporal/query`
- 轮询间隔：3 秒

**日志数据获取**：

- 读取 `audit-logs/{sessionId}/agents/` 目录
- 监听文件变化或定时刷新
- 日志格式：JSON，每行一条记录

### 3.3 响应式设计

- 桌面端：三栏布局
- 平板端：三栏自适应，收窄侧边栏
- 移动端：单栏布局，底部标签切换
