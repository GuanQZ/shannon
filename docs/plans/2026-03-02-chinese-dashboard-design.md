# 中文渗透测试监控仪表板设计文档

**项目**：Shannon 渗透测试框架
**日期**：2026-03-02
**功能**：中文实时监控仪表板（含 WebSocket 翻译服务）

---

## 1. 概述

本文档描述了为 Shannon AI 渗透测试框架设计的中文实时监控仪表板，包含独立的 WebSocket 翻译服务。

### 1.1 设计目标

- **实时性**：通过 WebSocket 实现日志实时推送，延迟 < 1秒
- **可读性**：全中文界面，保留专业术语英文
- **独立性**：翻译服务独立于 Temporal，不修改现有代码
- **高效性**：过滤无用日志，只翻译关键内容

### 1.2 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 翻译服务 | Node.js + chokidar | 监听文件变化，实时翻译 |
| 实时通信 | WebSocket (ws) | 推送翻译后的日志到前端 |
| 前端 | 原生 HTML/CSS/JS | 单页面应用 |

## 2. 界面设计

### 2.1 整体布局

采用三栏式布局，从左到右依次为：阶段进度栏 → 实时日志栏 → Agent 列表栏。

```
┌─────────────────────────────────────────────────────────────────────┐
│  Shannon 渗透测试监控                                               │
├────────────┬────────────────────────────────────┬───────────────────┤
│            │                                    │                   │
│  阶段进度  │         实时日志                   │    Agent 列表     │
│            │                                    │                   │
│  ● Pre-Recon │ [pre-recon] 正在执行子域名扫描   │  ▸ Pre-Recon     │
│  ● Recon    │ [pre-recon] 识别到 15 个端点      │    pre-recon     │
│  ● Vuln    │ [recon] 开始端口扫描               │  ▸ Recon         │
│  ● Exploit │                                    │    recon          │
│  ● Report  │                                    │  ▸ Vulnerability │
│            │                                    │    vuln-injection│
│            │                                    │    vuln-xss      │
│            │                                    │  ▸ Exploitation  │
│            │                                    │  ▸ Reporting     │
│            │                                    │                   │
└────────────┴────────────────────────────────────┴───────────────────┘
```

### 2.2 左侧：阶段进度条

**功能**：显示五个渗透测试阶段的执行状态

**阶段列表**：

1. **Pre-Recon**（信息收集前置）
2. **Recon**（侦察）
3. **Vulnerability**（漏洞分析）
4. **Exploitation**（漏洞利用）
5. **Reporting**（报告生成）

**视觉设计**：

- 每个阶段用圆形标记表示状态
- 状态颜色语义：
  - 灰色：未开始
  - 黄色闪烁：执行中
  - 绿色：已完成
  - 红色：失败
- 阶段名称使用中文
- 当前阶段显示"执行中"文字提示

**布局参数**：

- 宽度占比：约 10%
- 垂直排列，居中对齐

### 2.3 中间：实时日志栏

**功能**：显示所有 Agent 的实时执行日志

**日志格式**：

```
[agent-name] 日志内容
```

其中 `agent-name` 为 Agent 名称前缀，用于区分不同 Agent 的输出。

**功能特性**：

- **自动滚动**：新日志自动滚动到底部
- **中文翻译**：日志内容保持原样（已为中文）
- **AI 思考过程**：显示 AI 推理过程（thinking 内容）
- **时间戳**：每行日志可选择显示时间戳
- **颜色编码**：不同级别日志用不同颜色（info: 白色，warning: 黄色，error: 红色）

**数据源**：

- 方案一：轮询 Temporal 查询接口获取进度
- 方案二：直接读取 `audit-logs/` 目录下的 JSON 日志文件

**布局参数**：

- 宽度占比：约 60%
- 最大高度：视窗高度减去标题栏
- 滚动：支持鼠标滚轮和滚动条

### 2.4 右侧：Agent 列表栏

**功能**：按阶段分组的 Agent 列表，支持点击筛选

**分组结构**：

```
▸ Pre-Recon (1)
    pre-recon
▸ Recon (1)
    recon
▼ Vulnerability (5)
    vuln-injection ✓
    vuln-xss
    vuln-auth
    vuln-authz
    vuln-ssrf
▸ Exploitation (5)
▸ Reporting (1)
```

**交互设计**：

- 点击阶段名称：折叠/展开该阶段下的 Agent 列表
- 点击 Agent 名称：筛选日志，只显示该 Agent 的日志
- 再次点击同一 Agent：取消筛选，显示所有日志
- 选中状态：Agent 名称高亮显示
- 完成标记：已完成的 Agent 显示 ✓ 绿色勾选

**布局参数**：

- 宽度占比：约 30%
- 每个 Agent 占用一行
- 折叠时只显示阶段名和数量

## 3. 功能规格

### 3.1 日志过滤规则

| 类型 | 保留？ | 说明 |
|------|--------|------|
| `agent_start` | ✅ | Agent 启动 |
| `agent_end` | ✅ | Agent 结束 |
| `llm_response` → `thinking` | ✅ | AI 思考过程 |
| `llm_response` → 普通文本 | ✅ | AI 输出（人类可读） |
| `llm_response` → `tool_use` | ❌ | 工具调用（不显示） |
| `tool_result` | ❌ | 工具返回结果（不显示） |

**预期效果**：原始 6500 条日志 → 过滤后约 500-800 条（减少 87%）

### 3.2 翻译规则

- **专业术语**：保留英文（CVEs, payloads, endpoints, HTTP methods, file paths）
- **叙事内容**：翻译为简体中文
- **格式**：`[agent-name] 翻译后的内容`

### 3.3 WebSocket 协议

**服务端推送**：
```json
{
  "type": "log",
  "timestamp": "2026-03-02T14:07:42.264Z",
  "agent": "pre-recon",
  "subType": "text|thinking|agent_start|agent_end",
  "original": "I will analyze the security...",
  "translated": "我将分析安全..."
}
```

**前端连接**：
```
ws://localhost:3457?session=host-docker-internal_shannon-xxxxx
```

### 3.4 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 阶段进度显示 | 实时显示 5 个阶段的执行状态 | 必选 |
| Agent 列表显示 | 按阶段分组显示 13 个 Agent | 必选 |
| 日志实时显示 | WebSocket 接收翻译后的日志 | 必选 |
| Agent 筛选 | 点击 Agent 筛选对应日志 | 必选 |
| 筛选切换 | 重复点击取消/启用筛选 | 必选 |
| AI 思考显示 | 显示 AI 推理过程 | 必选 |

### 3.5 数据交互

**翻译服务**：

- 启动命令：`node dashboard/server.js --session <workflow-id>`
- 监听 `audit-logs/{session}/agents/*.log` 目录
- 文件变化时读取新行 → 过滤 → 翻译 → WebSocket 推送

**前端连接**：

- 连接：`ws://localhost:3457`
- 参数：`?session=workflow-id`
- 自动重连机制

### 3.6 响应式设计

- 桌面端：三栏布局
- 平板端：三栏自适应，收窄侧边栏
- 移动端：单栏布局，底部标签切换
