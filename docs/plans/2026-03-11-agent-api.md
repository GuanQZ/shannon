# 内网 Agent Chat API 接口文档

> 更新时间：2026-03-11
> 来源：实际接口截图

---

## 一、接口概览

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| init_session | POST | `/agent-api/{agent-id}/chatabc/init_session` | 初始化会话 |
| chat | POST | `/agent-api/{agent-id}/chatabc/chat` | 发送聊天请求 |

> **注意**: URL 路径包含 `{agent-id}` 占位符，如 `tool-agent-1-fa8496f6`

---

## 二、请求头

### Headers

```
Content-Type: application/json
Cookie: agt_token={jwt_token}
```

| 请求头 | 必填 | 说明 |
|--------|------|------|
| `Content-Type` | 是 | `application/json` |
| `Cookie` | 是 | 包含 JWT 认证令牌 `agt_token` |

> **说明**: JWT token 可以是固定值（用于测试/开发），mock-server 接收请求时不强制验证。

---

## 三、init_session 接口

### 3.1 请求

**URL**: `POST /agent-api/{agent-id}/chatabc/init_session`

**Body**:
```json
{
  "appId": "mock-app-id",
  "trCode": "mock-tr-code",
  "trVersion": "1",
  "timestamp": 1234567890,
  "requestId": "mock-request-id",
  "data": {
    "prompt_variables": []
  }
}
```

**字段说明**：
| 字段 | 类型 | 必填 | 说明 | 测试值 |
|------|------|------|------|--------|
| `appId` | string | 是 | 应用 ID | `mock-app-id` |
| `trCode` | string | 是 | 交易代码 | `mock-tr-code` |
| `trVersion` | string | 是 | 交易版本号 | `1` |
| `timestamp` | number | 是 | 时间戳 (毫秒) | `1234567890` |
| `requestId` | string | 是 | 请求唯一 ID (UUID) | `mock-request-id` |
| `data.prompt_variables` | array | 否 | 提示词变量数组 | `[]` |

> **说明**: 所有字段均可填充固定值（测试用）。

### 3.2 响应

```json
{
  "resCode": "FAIAG0000",
  "resMessage": "SUCCESS",
  "responseId": "resp_xxx",
  "timestamp": 1234567890,
  "data": {
    "session_id": "session_xxx"
  }
}
```

## 四、chat 接口

### 4.1 请求

**URL**: `POST /agent-api/{agent-id}/chatabc/chat`

**Body**:
```json
{
  "appId": "mock-app-id",
  "trCode": "mock-tr-code",
  "trVersion": "1",
  "timestamp": 1234567890,
  "requestId": "mock-request-id",
  "data": {
    "session_id": "session_xxx",
    "txt": "prompt 内容",
    "stream": true,
    "files": []
  }
}
```

**字段说明**：
| 字段 | 类型 | 必填 | 说明 | 测试值 |
|------|------|------|------|--------|
| `appId` | string | 是 | 应用 ID | `mock-app-id` |
| `trCode` | string | 是 | 交易代码 | `mock-tr-code` |
| `trVersion` | string | 是 | 交易版本号 | `1` |
| `timestamp` | number | 是 | 时间戳 | `1234567890` |
| `requestId` | string | 是 | 请求唯一 ID | `mock-request-id` |
| `data.session_id` | string | 是 | 从 init_session 获取 | 动态值 |
| `data.txt` | string | 是 | 用户输入（提示词） | **动态值** |
| `data.stream` | boolean | 是 | 流式输出 | `true` |
| `data.files` | array | 否 | 文件列表 | `[]` |

> **说明**: 除 `data.txt`、`data.session_id`、`data.stream` 外，其他字段可填充固定值（测试用）。

### 4.2 响应 (SSE 流式)

#### chat_started 事件

```
event: chat_started
data: {"chat_id": "abc123"}
```

#### message 事件 - AI 消息

**包含工具调用**:
```
event: message
data: {
  "content": "文本内容...（包含下一步操作的说明）",
  "additional_kwargs": {},
  "response_metadata": {},
  "type": "AIMessageChunk",
  "name": null,
  "id": "run-xxx",
  "example": false,
  "tool_calls": [
    {
      "name": "save_deliverable",
      "args": {
        "deliverable_type": "CODE_ANALYSIS",
        "content": "..."
      },
      "id": "call_xxx",
      "type": "tool_call"
    }
  ],
  "invalid_tool_calls": [],
  "usage_metadata": null,
  "tool_call_chunks": [
    {
      "index": 0,
      "id": "call_xxx",
      "name": "save_deliverable",
      "type": "tool_call_chunk",
      "args": "{\"deliverable_type\": \"CODE_ANALYSIS\", \"content\": \"...\"}"
    }
  ]
}
```

> **说明**: `tool_calls` 中每项的 `type` 为 `"tool_call"`；`tool_call_chunks` 中每项的 `type` 为 `"tool_call_chunk"`。

**纯文本**:
```
event: message
data: {
  "content": "这是一个文本回复...",
  "additional_kwargs": {},
  "response_metadata": {},
  "type": "AIMessageChunk",
  "name": null,
  "id": "run-xxx",
  "example": false,
  "tool_calls": [],
  "invalid_tool_calls": [],
  "usage_metadata": null,
  "tool_call_chunks": []
}
```

> **说明**: 字段 `additional_kwargs`、`response_metadata`、`invalid_tool_calls`、`usage_metadata` 固定为空值，`name` 固定为 `null`，`example` 固定为 `false`。

#### message 事件 - 工具执行结果

```
event: message
data: {
  "content": "工具执行返回的结果",
  "additional_kwargs": {},
  "response_metadata": {},
  "type": "function",
  "name": "save_deliverable",
  "id": null
}
```

> **说明**: 字段 `additional_kwargs`、`response_metadata` 固定为空对象，`id` 固定为 `null`。

#### done 事件

```
event: done
data: {"code": "FAIAG0000", "success": true}
```

## 五、工具调用处理流程

> **注意**: 工具调用循环由 Claude Agent SDK 内部处理（通过 `maxTurns` 配置），mock-server 是格式适配层。

### 5.1 SDK 消息类型

Claude Agent SDK 的 `query()` 函数返回以下流式消息类型：

| SDK 消息类型 | 说明 |
|-------------|------|
| `assistant` | AI 助手文本消息（可能包含 text content block） |
| `tool_use` | 工具调用请求（独立的工具调用块） |
| `tool_result` | 工具执行结果 |
| `result` | 最终结果（SDK 完成任务） |
| `system` | 系统消息（如模型初始化信息） |

> **注意**: `tool_use` 是独立的消息类型，不是 `assistant` 消息内的 content block。

### 5.2 mock-server 转换逻辑

**SDK 多轮循环**：SDK 内部会进行多轮工具调用，每轮流程：
```
assistant (文本) + tool_use (工具调用) → SDK 自动执行工具 → tool_result → 继续下一轮...
```

**SSE 消息对应**：
```
SDK 消息类型          →  SSE 消息 (一轮完整交互)
─────────────────────────────────────────────────────
assistant + tool_use  →  message (AIMessageChunk, 包含 tool_calls)
tool_result           →  message (function, 工具结果)
result                →  message (AIMessageChunk, 最后文本) → done
```

> **关键**: 一个 `message` 事件包含一轮完整的交互（文本 + 工具调用），而不是单独发送。工具结果单独作为 `function` 类型发送。

### 5.3 完整流程

```
1. 收到 chat 请求 (带 session_id + prompt)

2. 通过 Claude Agent SDK 调用 Claude Agent Server
   → SDK 内部配置了 MCP Servers (lumin-helper, playwright)

3. SDK 多轮循环处理:
   轮次 N:
   - assistant: 提取文本内容
   - tool_use: 提取工具调用
   → 组合为 message (AIMessageChunk) 发送
   → SDK 自动通过 MCP 执行工具
   - tool_result: 提取结果
   → 发送 message (function)
   → 继续下一轮...

4. SDK 返回 result (最终文本)
   → 发送 message (AIMessageChunk)
   → 发送 done 事件
```

> **流式处理**: SDK 的 `query()` 函数是流式响应的，即时的工具调用和文本内容会实时返回。mock-server 实时处理这些流式输出，**按轮次组装**后转换为 SSE 格式返回给 Lumin。
